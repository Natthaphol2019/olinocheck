# 🔄 สรุปการปรับปรุงระบบกะงาน (Shift System Refactor)

## 📌 ปัญหาเดิม

1. **พนักงานต้องเลือกกะงานทุกครั้ง** ที่เช็คอิน - ไม่มี auto-select
2. **ไม่มีการตรวจจับสายอัตโนมัติ** - status เป็น 'normal' เสมอ
3. **หน้าจัดการกะงานใช้งานยาก** - UI ไม่ชัดเจน
4. **พนักงานดูเวลากะตัวเองไม่ได้** - ไม่มีหน้าแสดงข้อมูลกะ
5. **HR ต้องจัดกะให้พนักงานทีละคน** - ไม่มี batch assign

---

## ✅ สิ่งที่ปรับปรุง

### 1. **ระบบตรวจจับสายอัตโนมัติ** 🚨

สร้าง PostgreSQL Trigger ที่:
- ✅ เปรียบเทียบเวลาเข้างานกับเวลาเริ่มกะ
- ✅ คำนวณ grace period (เวลาที่สายได้ เช่น 15 นาที)
- ✅ ตั้ง status เป็น `'late'` หรือ `'normal'` อัตโนมัติ
- ✅ ทำงานทั้ง INSERT และ UPDATE

**ตัวอย่าง:**
```
กะเช้า: เริ่ม 09:00
Grace period: 15 นาที

เข้างาน 09:10 → ปกติ ✅
เข้างาน 09:20 → สาย ❌
```

**ไฟล์:** `shift_system_improvement.sql`

---

### 2. **Auto-Select กะงานให้พนักงาน** ⚡

- ✅ โยกะงานที่พนักงานถูกจัดไว้จาก `employee_shifts`
- ✅ เลือกอัตโนมัติเมื่อเปิดหน้า Check-In/Out
- ✅ เปลี่ยนกะได้ถ้าต้องการ (แต่แนะนำกะให้แล้ว)
- ✅ ลดขั้นตอนจาก 3 เหลือ 1 ขั้น

**โค้ด:**
```javascript
// src/pages/CheckInOut.jsx
const loadEmployeeShift = async () => {
  // โหลดกะที่จัดให้พนักงาน
  const { data } = await supabase
    .from('employee_shifts')
    .select('*, shifts:shift_id (*)')
    .eq('employee_id', employee.id)
    .eq('is_active', true)
    // ...
  
  if (data?.shifts) {
    setSelectedShift(data.shifts.id) // Auto-select!
  }
}
```

## 🎯 วิธีใช้งาน

### **สำหรับ HR/Admin:**

#### **1. สร้างกะงานใหม่:**
```
1. เข้าหน้า "จัดการกะงาน" (/manager/shifts)
2. คลิก "สร้างกะงาน"
3. กรอกข้อมูล:
   - ชื่อกะ (เช่น กะเช้า, กะบ่าย)
   - เลือกสี
   - เวลาเข้า-ออก
   - เวลา OT (ถ้ามี)
   - Grace period (นาที)
4. บันทึก
```

#### **2. จัดกะงานให้พนักงาน (หลายคน):**
```
1. คลิก "จัดกะให้พนักงาน"
2. เลือกพนักงาน (tick ได้หลายคน)
3. เลือกกะงาน
4. เลือกวันที่เริ่มต้น
5. บันทึก → จัดกะให้ทุกคนที่เลือก
```

#### **3. แก้ไข/ปิดกะงาน:**
```
1. ที่ Card กะงาน
2. คลิกไอคอน Settings → แก้ไข
3. คลิกไอคอน X → ปิดใช้งาน
```
